# gr-scheduler 受入テスト手順書 (Acceptance Test Procedure / UAT)

- 対象: gr-scheduler MVP（user-order 62項目）
- 目的: ユーザーがブラウザで主要機能を確認し受入判定する

## 前提・起動

- 開発サーバ: リポジトリで `npm install`（初回のみ）→ `npm run dev` → ブラウザで `http://localhost:5173/`
- または単一HTML: `npm run build` 後、`dist/index.html` をブラウザで直接開く（サーバー不要）
- 自動チェック（任意）: `npm run test`（単体213）・`npx playwright test`（E2E/a11y 4）・`npm run coverage`

各項目: 操作 → 期待結果 →（合否）。UC/要求UIDを併記。

## A. 俯瞰・表示・ズーム（UC-001/004）

| # | 操作 | 期待結果 | 合否 |
