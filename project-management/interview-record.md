## 1. プロジェクトの狙い（Why）

`old/日程管理ツール.md` の結論「マルチバー要件（1行に複数バー）を満たす Web 型ツールは
調査範囲に存在せず、選択肢は iQUAVIS 購入 / MS Project / 自社構築」の**自社構築案を実行**する。
パワポ/Excel での日程表作成の非効率を、WYSIWYG かつ構造化データ（JSON/XML）を持つ
単一HTMLツールで置換する。

## 2. 確定した重要判断（ユーザー回答）

| 論点 | 決定 |
|---|---|
| 技術スタック | TypeScript + Vite + `vite-plugin-singlefile`（単一HTML成果物）。UIは vanilla TS + SVG |
| 仕様形式 | StrictDoc（`.sdoc` + `.sgra`）。`StrictDocStarter/samples/sovd-automotive-ja` を模範 |
| 製品名 | `gr-scheduler`（リポジトリ名のまま） |
| データ保存/読込 | ファイル入出力（.json/.xml）+ localStorage 自動保存（クラッシュ復旧） |
| MSProject連携 | MSPDI XML 双方向 Import/Export（.mpp バイナリは対象外） |
| 想定規模（性能NFR基準） | 中規模: ~50行 / ~1000アイテム。60fps のズーム/パン、初期表示 1.5s 以内 |
| MVP スコープ | user-order item1-58 のコア機能。item59（将来拡張）は設計で拡張点のみ確保し実装外 |
| 条件付きプロセス | WCAG 2.1 AA と 製品i18n/l10n を有効化。他は無効 |

## 3. ドメイン境界の識別

**コアドメイン（自作の中心）:**
1. マルチバー・レイアウトエンジン（1行複数アイテム・重なり回避・上下左右整列）
2. 異方性ズーム + LOD（縦/横独立ズーム・時間軸粒度自動切替・アイテム自動増減）
3. 依存線の自動配線（9点アンカー・重なり最小化・折れ点0〜3）
4. 時間↔座標マッピング（日付/期間と画面座標の相互変換）
5. イナズマ線 / 予実差分（実績遅延の可視化・変更前予定グレー表示）
6. データ形式変換（内部モデル ↔ JSON ↔ MSPDI XML）

**既存技術の利用（Adapter/Framework、ドメインではない）:**
SVGレンダリング、File API、localStorage、絵文字/フォント描画、SVG/PNG インポートの解析基盤。

## 5. エッジケース・制約（精読で抽出、設計で扱う）

- 同一タイミングのタスク/同種マイルストーンを同じ高さに揃える（item3）→ 整列制約ソルバ
- 略称表示位置をドラッグ移動（item22）、丸角囲みの R はズーム非依存で一定（item32.1）→ スクリーン空間固定要素
- ズームでアイテムを自動増減（item47）→ 重要度（priority/weight）に基づく LOD 間引き。「重要は縮小でも残す/細部は拡大で消える」
- ヘッダー・分類エリアはスクロールしても常時表示（item49）、分類エリアとスケジュールエリアは連動スクロール（item40）→ 固定ペイン + 同期スクロール
- 透かし（item58）: ログインユーザー名（＝クライアントのため「ローカル入力ユーザー名」）+ 日時、斜めタイル、薄さ、表示切替
- パレット色は色弱者対応で原色不使用（item20）→ CUD 準拠パレット（白/黒/濃灰/薄灰/赤/青/黄/緑/橙/紫の低彩度版）
- property 多国語（item17）だが property 名は英語固定（item18）→ i18n はラベル層、キーは英語

## 6. item61（追加便利機能）の提案候補（設計で採否判断）

パワポ日程表を代替する際にあると便利な機能（MVP採否は architect/ユーザーで判断）:
- スナップ整列（近接アイテムへのマグネット吸着・ガイドライン表示）
- 期間の相対編集（端ドラッグでリサイズ、本体ドラッグで平行移動、Shiftで日単位スナップ）
- テンプレート/スタイルのコピー（書式のみ貼付）
- 祝日・非稼働日の網掛け（ロケール別カレンダー、i18n と連動）
- クリティカルパス強調（依存関係からの自動算出）※将来
- PNG/クリップボード出力（SVGに加えて会議貼付用）
- 印刷/PDF レイアウト（A3横 等の一枚絵出力）
- ミニマップ（全体俯瞰 + 現在ビュー枠）
- Undo/Redo とショートカット（item52）
- アクセシビリティ: キーボードのみでの作成・編集経路
