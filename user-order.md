# ユーザーオーダー

## 何を作りたい？

1つの.htmlでブラウザだけで動作する日程表作成ツール
パワポで日程表を書く作業とほとんど同じUIだが、画像データだけではなく、JSON/XMLデータを持つツール

## それはどうして？

パワポやExcelで日程表を書くのは非効率
MSProjectとかIQUAVISとかは使いにくい。
C:\Users\good\_\OneDrive\Documents\GitHub\gr-scheduler\old\日程管理ツール.md
で調べたほかのツールも1行に複数のマイルストーンやタスクを入れられないから、使いにくい。

## その他の希望

1. プロジェクトの全体日程を1つの画面でスクロールなしに見れること
2. 1行に複数のマイルストーンやタスクを入れられること
3. マイルストーンやタスクとそのアイコンは上下左右揃えをできること (同じタイミングでタスクを始める/同種のマイルストーンは同じ高さに表示する)
4. 1つの.htmlだけで動作すること
5. WYSIWYGなUIであること
6. ★★スケジュール画面をいっぱいに使えるようにすること (狭い画面に最大の情報量)
   1. ★★ヘッダー領域は視認できる範囲で最小化する
   2. ★★ツールパレットはフローティングとする
   3. ★★フォントサイズを全体で変更可能とすること (大/中/小)
   4. 初見の各国ユーザーがマニュアルを見ないで使えること
   5. アフォーダンスにこだわること
   6. 各種メニューは言葉ではなくアイコンで用途を伝えること
7. デフォルトでマイルストーンやタスクのアイコンを提供すること
8. デフォルトのアイコンは以下を提供すること
9. マイルストーン
   1. 〇△▽□☆5角形六画形
   2. タスク
   3. ー
   4. →
   5. ⇔
   6. _-_
   7. 矢羽根 |＞
10. アイコンには絵文字を使えること
11. アイコンは枠の色と塗りつぶしの色を選択できること
12. ★アイコンの移動に連動してアイコンのプロパティーが自動で変わること
13. マイルストーンやタスクをコピペできること (類似の作業の入力が楽)
14. マイルストーンやタスクのアイコンをインポートできること
15. インポート形式はSVGおよびPNGとする
16. マイルストーンやタスクにプロパティーを設定できること
17. マイルストーンやタスクのプロパティーは以下を含むこと
    1. 略称
    2. 分類
    3. アイコン用絵文字/記号
    4. 開始日
    5. 終了日
    6. 予定 or 実績の区別
    7. 表示位置
    8. 自動
    9. 中心
    10. 上
    11. 下
    12. 右
    13. 左
    14. 枠色 (フルカラー/パレット)
    15. 塗りつぶし色 (フルカラー/パレット)
    16. 線の太さ (細/中/太)
    17. 正式名称
    18. 説明
    19. 大分類
    20. 中分類
    21. 小分類
    22. 担当
    23. ステータス
    24. 備考
18. propertyは多国語対応すること
19. property編集時のproperty名は英語表記とする
20. パレット色は白/黒/濃い灰色/うすい灰色/赤/青/黄色/緑/オレンジ/紫/★★★透明を選択可能なこと
21. パレット色は色弱者対応として原色を使わないこと
22. マイルストーンやタスクの内部or付近には略称を表記可能なこと
23. 略称の表示位置をマウスドラッグで移動できること(マイルストーンやタスクが狭いところに込み合っても見やすい)
24. スケジュールのヘッダー左上にスケジュール名を表示できること
25. スケジュールのヘッダーに年/月/日/曜日を表示できること
26. スケジュールの拡大縮小で年/月/日/曜日の表示/非表示が自動で切り替わること
    1. 年だけ
    2. 年/月
    3. 月/日/曜日
27. 本日の線を表示できること
28. 任意の縦線を2つ表示して(デュアルカーソル)スパンを測定できること (タスクAからタスクBまで何日か測る)
29. デュアルカーソルは縦線モードと十字線モードを個別に切り替えできること(どの分類のタスクが何日か見やすくする)
30. 本日の線やデュアルカーソルは表示/非表示を切り替えできること
31. 任意の場所にコメントを記載できること
32. コメントは下記2種を提供すること
    1. 引出し四角: -□ とか /□
    2. 折れ線: /---- とか \_\_\_\_/ とか
33. 任意の場所を任意の色の各丸□で囲えること(一部を強調)
    1. 各丸四角のRは日程を拡大縮小しても同じ大きさで見えること(角を拡大しても〇のサイズが大きくならない)
34. タスク/マイルストーンの依存関係を表現できること
    1. アイコンの上下左右角と中央(9点)から引き出し/引き入れできること
    2. 依存関係線は他のアイコンと極力かぶらないように自動で引けること
    3. 依存関係線の折れ曲がり可能な点は0～3とすること
35. タスク/マイルストーンを分類する水平線を引けること
36. ★分類は複数の行を束ねてセクションとし、セクション単位で順序を入れ替え可能なこと
37. ★各セクションは表示/非表示を選択可能なこと
38. ★非表示のセクションは水平線の左端にある小タブで再表示可能なこと
39. ★非表示のセクションが増えた場合は水平線の小タブを増やすこと
    1. ★水平線の幅を太くしない
40. タスク/マイルストーンの分類名をスケジュール画面の左側に記載できること
41. 分類エリアとスケジュールエリアは個別に連動してスクロール可能なこと (拡大した時でも分類エリアが画面左に残る)
42. MSProjectの日程データ(.xml?)のImport/Exportができること。
43. JSONで日程データをImport/Exportできること。 (AI向け)
44. 画面をSVGで出力できること
45. マウスのセンターホイールで拡大縮小ができること
46. ★縦方向のみ拡大と横方向のみ拡大ができること
47. 拡大縮小によって年単位のスケジュールから日単位のスケジュールまで表示できること
48. 拡大縮小によって表示するアイテムが自動的に増減すること。 (重要なマイルストーンやタスクは縮小しても表示する。細かいマイルストーンやタスクは拡大で消える。)
49. 拡大したら上下左右にスクロールできること
50. 上下左右にスクロールしてもヘッダー領域と分類領域は常に表示すること
51. 初期表示としてのテンプレートを1つ提供すること
52. 拡大した場合、マウス操作だけでもスクロールできること
53. ショートカットキーを使えること
54. 予定と実績を表示/管理できること
55. 予定のみ表示/実績のみ表示/予定と実績両方表示ができること
56. 実績表示の場合、イナズマ線を引けること(どこがボトルネックか可視化する)
57. 変更前の予定をグレーで表示し、計画の変更を可視化できること
58. ツール操作用パレットを選択していないときは薄く透明表示とすること
59. 日程表に透かしを入れられること (Web会議で秘密の日程を宿所されたときにエビデンスを残す)
    1. 透かしはログインユーザー名と日時を入れること
    2. 透かしはギリギリ見えるくらいの薄さで画面全体に斜め /方向で複数入れること
    3. 透かしの表示/非表示は切り替えできること
60. 将来は以下の機能を拡張できる設計としておくこと
    1. サーバーおよびブラウザ間E2E暗号通信で共同編集可能
    2. ユーザーごとに行の公開と編集権限を設定可能
       1. 上層部が決めたスケジュールを参照できるが編集できないようにする
       2. 自分のプロジェクトを見れる人を限定可能
    3. 複数のプロジェクトを統合
    4. 1つのプロジェクトからサブプロジェクトを作成
    5. ガントチャートモードと一覧モード(要命名)を切り替え可能なこと
    6. ガントチャートモードではタスク大中小分類/タスク/担当/期限/ステータス/備考の列を設けること
61. 各種命名は名は体を表すこと(ソフトは言霊)
62. その他パワポでの日程表作成を本ツールで代替するときにあると便利な機能があれば提案すること
