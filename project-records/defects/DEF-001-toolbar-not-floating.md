# DEF-001: メインツールバーが固定・肥大でフローティングしない

- 発見: 2026-07-19（納品判定 GO 後、ユーザーの実機スクリーンショットで発覚）
- 重大度: High（★★要求の非準拠 / UX中核）
- 状態: **修正済み（検証グリーン）**
- 関連要求: STK-L0-021, TOOL-L1-001, TOOL-L1-006, item6 / 6.1 / 6.2 / 6.6

## 事象
`src/app/main.ts` の `buildChrome()` がコマンド操作群を flex-column の第1子＝**固定ヘッダー**として配置。
キャンバス上部を大きく占有し、`nowrap` 1行に長いデバッグ見出し＋約14の冗長テキストボタンを詰め込み、
見出しが最小幅に縮んで多行折返し→帯が肥大。フローティングせず、item6/6.1/6.2 に違反。

## 根本原因
1. 実装がモック（`docs/mock/gr-scheduler-mock.html`＝正しい設計）と乖離。M5b のフローティング化は
   図形パレット（`tool-palette.ts`）のみ対象で、メインのコマンドツールバーが対象外だった。
2. **レビューがレイアウトの視覚検証をしていなかった**（R2-R5/axe はロジック・セキュリティ・a11y中心で、
   ★★の見た目=フローティング/最小化を screenshot で確認していない）→ 見逃し。

## 対策（実装）
- 固定ツールバー撤去、キャンバスをビューポート全面化。
- コマンド操作を `position:absolute` の**フローティング半透明パレット**（背景alphaのみ透過・文字不透明、
  hover/focus-withinで不透明）へ移動。`flex-wrap` + `max-width` で肥大防止。
- ヘッダーは**スケジュール名のみの最小帯**（≤26px）、デバッグ見出し削除。
- ボタンをアイコン+aria-label/title化（item6.6・a11y維持）。
- 回帰ガード `tests/e2e/layout-mock-conformance.spec.ts` 追加（キャンバス上端/ヘッダー高さ/パレットの
  absolute配置・idle透過・hover/focus不透明を assert）。

## 検証
213単体PASS / lint・tsc 0 / 単一HTML 114.66kB(CSP) / Playwright 5 PASS（新ガード+axe AA含む）。

## 再発防止（プロセス改善提案）
- **レビュー観点に「視覚レイアウト検証（★★準拠のフローティング/最小化/全面化）」を追加**。
  UI変更を伴うマイルストーンでは screenshot ベースの目視 or レイアウト幾何のガードテストを必須化する。
- モックを「視覚仕様の基準」と明記し、実装のレイアウトがモックに準拠しているかを受入基準に含める。
