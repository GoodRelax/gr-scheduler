# DEF-010: ヘッダーのプロジェクトタイトルがストア変更に継続追従しない

- 起票: 2026-07-22（ユーザー指摘「プロジェクト名が出力されない」の調査中に発見）
- **訂正: 2026-07-22**（初版の記述が不正確だったため全面改訂。下記「初版の誤りと訂正」参照）
- 種別: defect（表示とモデルの同期漏れ）
- 重大度: **Low**（初版では Medium と評価したが、束縛自体は存在し、影響は「タイトルを変更する経路を
  追加したときに追従しない」に限られるため引き下げ）
- 状態: **Fixed（2026-07-22、実機ライブ検証済み）** — `bindHeaderTitleText` が `store.subscribe` を
  購読し、dispatch/undo/redo/replaceDocument を網羅。空欄フォールバックは
  `HEADER_TITLE_PLACEHOLDER` に一本化。実機で「タイトル編集→確定→Undo」でヘッダーが毎回追従することを確認。
- 関連: TOOL-L1-008（ヘッダー構成）, CR-016（タイトルのダブルクリック編集・MSPDI `Name` 出力）

## 初版の誤りと訂正（重要）

初版で「ヘッダーのタイトルは `'gr-scheduler'` のハードコードで `ScheduleDocument.title` と
**非連動**」と記述したが、これは**誤り**であった。実コードを精査した結果:

- `src/app/main.ts` の `syncScheduleName(chrome, store)` は
  `store.getDocument().title` を読み、**空文字のときだけ** `'gr-scheduler'` にフォールバックする。
- 起動時（`main.ts` 内 1 箇所）と JSON/MSPDI Import 時（同 1 箇所）に呼ばれており、
  タイトルは**実データに束縛されている**。
- 構築時の `scheduleNameLabel.textContent = 'gr-scheduler'` は初期値であって恒久的な上書きではない。

誤診の原因は、既定テンプレートの `title` が `'gr-scheduler template'`（`sample-data.ts`）であり、
空欄フォールバック文字列 `'gr-scheduler'` と酷似していたため、「固定文字列が出ている」ように
見えたことによる。**ユーザーの当初指摘「JSON/XML にプロジェクト名を出力すること」も、
調査の結果すでに実装済み**（`json-codec` は `title` を必須で入出力、`mspdi-codec` は `<Title>` を
出力・読込）であり、新規要求ではなかった。

本票は上記を踏まえ、**実際に残っている欠陥だけ**を記述するよう改訂する。

## 事象（実際の欠陥）

ヘッダーのタイトル表示が、**ストアの変更に継続的に追従しない**。
`syncScheduleName` は「起動時」「Import 時」という**2つの時点でしか呼ばれておらず**、
`ScheduleStore` の変更通知を購読していない。

したがって、タイトルを変更する経路を新たに設けた場合
（CR-016 のダブルクリック編集、およびその Undo/Redo）、
**明示的に再同期を呼ばない限りヘッダー表示が古いまま**になる。

副次的に、構築時の初期値 `'gr-scheduler'`（フォールバック文字列と同一のリテラル）が
最初の同期までの一瞬だけ表示されうる二重管理になっている。

## あるべき挙動

- ヘッダーのタイトル表示は `ScheduleDocument.title` の変更に**継続的に追従**する
  （ストアの変更通知に購読し、Undo/Redo を含むあらゆる変更経路で自動更新される）。
- 空欄時の代替表示のルールは 1 箇所に集約し、構築時リテラルとの二重管理を解消する。

## 修正方針

`syncScheduleName` をストアの変更通知（`ScheduleStore` の購読、CR-009 で追加した
`onContentChange` を含む）に結線し、点呼び出しを廃する。構築時リテラルは削除するか、
フォールバック定数を単一の情報源として共有する。CR-016（ダブルクリック編集）の実装と
一体で行うのが自然。

## 検証

- 起動時にテンプレートの `title`（`gr-scheduler template`）が表示されること。
- 別タイトルの JSON / MSPDI を Load したとき、その `title` に切り替わること。
- **タイトルを編集 → Undo → Redo したとき、ヘッダー表示が毎回追従すること**（本票の主眼）。
- 単体テスト（表示文字列がモデル由来であること・変更追従）＋実機 DOM で確認する。

## 教訓（プロセス）

defect 票を起票する際は、症状の観察だけでなく**該当コードの全呼出経路を確認**してから
原因を断定すること。本件は「初期値リテラル」を見て「非連動」と早合点した。
