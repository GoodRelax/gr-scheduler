# DEF-003: 仕様・記録で無内容な符丁「案H / Model H」を使用（命名規約 item60 違反）

- 発見: 2026-07-20（本セッション、ユーザー指摘によるレビュー）
- 重大度: Medium（品質規約違反。データ契約の正当性自体には影響しないが CLAUDE.md 命名規約の明示違反）
- 状態: **修正済み（本セッションで全廃・記述化）**
- 関連要求: CLAUDE.md「命名は言霊」（user-order item60）
- 関連文書: `change-request-001-20260719-230349.md`, `R-CR001-spec-review.md`,
  `traceability-matrix.md`, `DEC-003-cr-001-approved.md`（旧 `DEC-003-cr-001-approved-plan-h.md`）

## 事象

CR-001（予実データモデルの変更）の検討過程で採用した作業ラベル「案H」（および英語表記
「Model H」/「モデルH」）が、CR 本文・レビュー報告・トレーサビリティ・意思決定記録・ファイル名
（`DEC-003-cr-001-approved-plan-h.md`）に広く残置していた。

「案H」は「予実の持ち方の選択肢 A〜H のうち 8 番目」という**検討時の内部整理番号**に過ぎず、
それが何を意味するかを名前自体が一切語らない。CLAUDE.md は明示的に以下を規定する。

> **命名は言霊**（user-order item60）: `type`, `data`, `info`, `value` 等の無意味な汎用語は禁止。
> 名前は「それが何か」を一目で伝える。

「案H」はこの規約が禁止する「無内容な符丁」の典型であり、`type`/`data` 等と同様の違反に該当する。
放置すると、将来の読者（実装セッション・レビュー・ユーザー）が「H」が何を指すか文脈を遡らないと
理解できない状態が記録として固定化される。

## 根本原因

- CR-001 の起票時、検討過程の作業用ラベル（A〜H の比較検討メモ由来）をそのまま正式記録（CR 本文・
  レビュー・トレーサビリティ）へ持ち込み、正式採用時に記述的な名前へ置き換える工程が抜けていた。
- change-manager・review-agent とも「命名は言霊」を StrictDoc 要求文（識別子・UID）には適用して
  いたが、**プロセス記録内の自然文ラベル**への適用チェックが欠落していた。

## 対策（実装）

- 「案H」/「Model H」/「モデルH」を、記述名「**実績日フィールド方式**（実績を actualStart/actualEnd
  で保持する方式）」へ全面置換した:
  - `change-request-001-20260719-230349.md`（タイトル・本文・Footer 変更履歴に廃止注記を追加）
  - `R-CR001-spec-review.md`（観点欄・指摘本文・Round 2/3 の判定文・Footer）
  - `traceability-matrix.md`（節見出し・本文・表内容）
  - `DEC-003-cr-001-approved.md`（`DEC-003-cr-001-approved-plan-h.md` から改称。本文中の符丁も除去）
  - 参照元 `change-request-002-20260720-054132.md` の関連リンク・本文
- CR-001 本文に「検討時の作業ラベル『案H』は廃止」という一文のみを履歴として残置し、以降は記述名
  のみを用いる方針を明記（DEC-004 §決定-3 参照）。

## 検証

上記 5 ファイルを grep し「案H」「Model H」「モデルH」の残存が 0 件であることを確認。
`docs/spec/**`・`src/**`・スキーマは本 defect の対象外（別セッションの実装フェーズで
30-architecture.sdoc・12-properties-i18n.sdoc の旧語彙残置（R-CR001-spec-review.md Round 3 の
OBS-1/OBS-2）と併せて整流予定）。

## 再発防止（プロセス改善提案）

- 検討過程の作業ラベル（選択肢比較のための一時符丁: A案/B案/Opt1/Opt2 等）は、**正式な CR/決定
  記録へ昇格する時点で必ず記述的な名前に置き換える**ルールをプロセス規則
  （`process-rules/full-auto-dev-document-rules.md` 等）に追記することを推奨する。
- レビュー観点（review-standards）に「記録内の自然文ラベルが命名規約（item60）に反していないか」
  の軽量チェックを追加することを推奨する（本件は StrictDoc 要求 UID には適用済みだったが、CR/
  DEC 等のプロセス記録には未適用だった）。
