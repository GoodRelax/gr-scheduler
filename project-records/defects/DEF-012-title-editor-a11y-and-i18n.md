# DEF-012: 編集可能タイトルの role 欠落（WCAG 4.1.2）とプレースホルダの i18n 漏れ

- 起票: 2026-07-23（CR-016 の仕様改訂中に architect が発見。レビュー L-8 ほか）
- 種別: defect（アクセシビリティ要求違反＋i18n 要求違反）
- 重大度: **Medium**（本プロジェクトは WCAG 2.1 AA を有効な品質目標として掲げており、
  4.1.2 Name/Role/Value は AA の必須項目。支援技術利用者がタイトルを編集可能と認識できない）
- 状態: **Fixed（2026-07-23）** — 静的タイトルに `role="button"` を付与（編集中は role を外し終了時に復元。
  実 `<button>` にすると編集用 `<input>` が入れ子になり axe `nested-interactive` に触れるため）。
  アクセシブル名は `"<プロジェクト名> - <操作ヒント>"` で可視文字列が先頭＝WCAG 2.5.3 も満たす。
  プレースホルダと操作ヒントは `UI_LABELS` に単一定義化し ja を追加、`locale.onChange` で再描画。
  検証: `tests/header-model.test.ts` / `tests/cr016-project-title.test.ts` / `tests/e2e/a11y.spec.ts`。
- 関連: CANVAS-L1-004（`I18N: Yes`）, TOOL-L2-010（CR-016 で新設したタイトル編集要求）,
  PROP-L1-003（UI 言語切替）, CR-016, 品質目標「アクセシビリティ WCAG 2.1 AA」

---

## 事象 1 — role 欠落（WCAG 2.1 AA 4.1.2）

CR-016 で追加したヘッダーのプロジェクトタイトルは、ダブルクリックで編集に入る対話的要素で
あるにもかかわらず、実装は

```
<span data-role="schedule-name" tabindex="0">…</span>
```

であり、**`role` を持たない**。`tabindex="0"` によりフォーカスは受け取るが、支援技術には
「ただのテキスト」としてしか伝わらず、**編集できることが伝わらない**。
WCAG 2.1 の 4.1.2（Name, Role, Value）に反する。本プロジェクトは CLAUDE.md の品質目標で
WCAG 2.1 AA を有効化しているため、単なる改善ではなく**要求違反**である。

## 事象 2 — プレースホルダが英語ハードコード（i18n）

空タイトル時の代替表示は `HEADER_TITLE_PLACEHOLDER = '(untitled schedule)'` として
**英語リテラルで固定**されている。CLAUDE.md は「製品 UI の表示文字列は i18n 対象」と定め、
`CANVAS-L1-004` も `I18N: Yes` である。CR-016 が新設した `TOOL-L2-010` は
「プレースホルダを単一定義とする」ことは要求したが**多言語化は要求していない**ため、
仕様と実装の双方に穴が残っている。

（補足: 定数名や `data-role` は英語 ASCII のままでよい。対象は**画面に出る文字列**のみ。）

## あるべき挙動

1. タイトル要素は編集可能であることが支援技術に伝わること。具体的な実現方法は実装時に確定するが、
   候補は (a) `role="button"` + 「クリックで名称を編集」の意図を持つアクセシブル名、
   (b) 実際の `<button>` 要素にする、(c) 編集開始後の `<input>` に適切なラベルを与えたうえで
   静的表示側にも役割を与える、のいずれか。編集中の `<input>` にもアクセシブル名が必要。
2. 空タイトル時のプレースホルダが i18n 対象となり、`activeLocale` に追従すること
   （単一定義であることは維持する）。

## 修正方針

`TOOL-L2-010` に role 要求と i18n 要求を追記したうえで実装する
（仕様を先に直さないと、実装だけ進めて再び仕様/実装の乖離を生む）。
axe による自動チェック（既存の `tests/e2e/a11y.spec.ts`）に、タイトル要素が
role を持つことの検証を追加する。

## 検証

- axe スキャンで 4.1.2 関連の violation が出ないこと。
- キーボードのみでタイトルを編集開始・確定・取消できること（既存の Enter/F2/Escape 経路）。
- 言語トグルでプレースホルダ表示が切り替わること。
