## 1. 背景・目的

CR-003 でヘッダーの配置順序と各ボタンの意味（`TOOL-L1-008`）が確定し、CR-002 で予実の描画スタイル
（Overlap/Separate、`PLAN-L1-005`）がデータとして定義された。本 CR は、実際の操作性向上のため
ユーザーから追加で要求された、ヘッダーおよびフローティングパレットの UI コントロール 8 点を
1 パッケージとしてまとめる。いずれも「使ってみて気づいた操作性改善」であり、AI 側の技術的都合
（defect 修正・設計改善）ではなくユーザー起点の要求追加・変更のため、change-manager が管理する。

## 2. 変更内容

### Part 1 — [Fit] ボタン（ヘッダー左端に追加）

Fit（全体表示）操作は既存のフローティングパレット内（View グループ、`fitButton` / `wireFit`）に
既に実装済みだが、使用頻度が高いため、**ヘッダーのボタン群の一番左**（`GR Scheduler +(c)` より
さらに左）に `[Fit]` ボタンを新設する。機能自体（`renderer.fitToContent()` 相当）は変更しない。

**改訂対象**: `TOOL-L1-008`（ヘッダーの並び順表に `Fit` を先頭要素として追加）。

### Part 2 — [P] パレット表示/最小化トグル（ヘッダー左から2番目）

ヘッダーの **左から2番目**（`[Fit]` の次）に `[P]` ボタンを追加し、フローティングパレットの
表示/最小化を切り替える。既存のパレット内 `[-]`（`▁` / `minimizeButton` ・ `togglePaletteMinimized`）
と**双方向連動**させる。すなわち、パレット側の `[-]` で最小化した場合はヘッダーの `[P]` も
連動して不活性（色変化）表示になり、逆にヘッダー `[P]` から最小化した場合もパレット側 `[-]` の
表示状態（`aria-expanded` 等）が追従すること。

**目的**: パレット最小化時にユーザーがパレットの居所を見失う（「迷子になる」）問題の対策。

**改訂対象**: `TOOL-L1-008`（ヘッダー並び順表に `P` を第2要素として追加）。関連（新規詳細化）:
`TOOL-L1-001`（フローティングパレット）に、ヘッダー `[P]` とパレット `[-]` の状態同期という
L2 相当の制約を追加する。

### Part 3 — [SS] 出力先をクリップボードへ変更（CR-003 改訂）

CR-003 (`TOOL-L1-008` 順表の SS 行) では SS = 表示域（現在のビューポート）のスクリーンショットを
**PNG としてファイル出力（ダウンロード）** と定義していた。本 CR はこれを、**クリップボードへの
画像コピー**（`navigator.clipboard.write` による ClipboardItem 経由の画像コピー）に変更する。
用途は、現在表示中の画面をそのままパワーポイント等へ即座に貼り付けること。ファイルダウンロードは
行わない（Save ボタンの PNG/SVG 全体出力とは引き続き役割が別）。

**改訂対象**: `TOOL-L1-008`（SS 行の「意味 (機能実体)」列を「PNG ダウンロード」から「クリップボードへ
画像コピー」に書き換え）。ブラウザの Clipboard API 権限・非対応環境時のフォールバック（例:
ダウンロードへのフォールバックまたはエラー通知）は実装フェーズで確定する。

### Part 4 — 言語トグル [en][jp]（画面右上、末尾要素の左）

画面右上の末尾要素の左に `[en]` `[jp]` ボタンを追加し、ヘッダーの `[AI]` と `[?]`（Help）の
表示文言を日英で切り替える。`activeLocale`（`PROP-L1-003` の UI 言語切替の仕組み）と連動する形で
よい。**明記事項**: `[AI]` ボタンが行う「AI 向けプロンプトのコピー」は**プロンプト部分のみ和訳**し、
**JSON スキーマ部分は和訳しない**（AI 向け相互運用性のため原文＝英語のまま保つ）。

**改訂対象**: `TOOL-L1-008`（AI・? の表示文言が `activeLocale` に応じて切り替わる旨を追記）。
関連: `PROP-L1-003`（UI 言語切替の対象範囲にヘッダーの AI/Help 文言を含める旨の詳細化）。

> **注記（要確認）**: 本文中「画面右上 [X] の左」の `[X]` は現行仕様 (`TOOL-L1-008`) の並び順表に
> 存在しないボタンである（現行の最右要素は `?`＝Help）。実装時に `[X]` が何を指すか
> （ウィンドウクローズ等の未仕様化要素か、`?` の位置を指す言い換えか）を orchestrator 経由で
> ユーザーに確認する必要がある。本 CR では暫定的に「ヘッダー最右要素（現行 `?`）のすぐ左」と
> 解釈して影響分析している。

### Part 5 — イナズマ線トグル（パレット・既定を非表示へ変更）

フローティングパレットに、イナズマ線（progress line）の表示/非表示トグルを追加する。位置は
カーソルボタンの右。**既定値を非表示に変更**する（現状 `viewState.progressLineVisible` は
未指定時に表示＝真、`src/app/main.ts` の `isVisible: () => ... !== false` および
`progress-today-layer.ts` の判定が示す通り現行の既定は表示）。

**改訂対象**: `PLAN-L1-003`（イナズマ線を引ける旨の要求に、パレットからの表示/非表示トグルと
既定値=非表示を追記）。データ既定値の記述がある `40-data-format.sdoc`（`progressLineVisible`
の既定値）および `docs/api/gr-scheduler.schema.json` も実装フェーズで整合させる。

### Part 6 — 予実スタイルトグル（パレット、[Ao]/[As]、既定は現状維持）

パレットに、Overlap/Separate（`PLAN-L1-005`、CR-002 で確定）を切り替える UI を追加する。
表記案: `[P][Ao][As]`（`Ao` = Overlap、`As` = Separate）。**既定は `[Ao]`（Overlap）のまま変更しない**
（`PLAN-L1-005` の既定値を据え置き、UI コントロールを新設するのみ）。

**改訂対象**: `PLAN-L1-005`（データとしては既存のまま。UI 操作面の新設を追記する L2 相当の詳細化）。

### Part 7 — 担当者名 表示/非表示トグル（パレット）

パレットに、アイテムの担当者名（`PROP-L1-002` の `assignee` フィールド）の行上表示 有無を
切り替えるトグルを追加する。表示時のレイアウト規則（配置位置等）は **CR-004 Part 5** に従う
（参照のみ。本 CR では新規に定めない）。

**改訂対象**: `PROP-L1-002` および `11-items-icons.sdoc`（アイテム描画に `assignee` の可視状態を
追加する旨の詳細化）。

> **注記（要確認）**: `project-records/change-requests/` 配下に **CR-004 のファイルが存在しない**
> （現存するのは CR-001/002/003 のみ）。CR-004 Part 5 の「担当者名表示時のレイアウト規則」が
> 別途どこかで決定済みかどうかを orchestrator 経由で確認する必要がある。未記録であれば、本 Part 7
> の実装前に CR-004（または本 CR への統合）としてレイアウト規則を別途確定させること。

### Part 8 — Add Box の位置指定（パレット、2クリック矩形指定）

現状、丸角囲み（`CURS-L1-007` Rounded-Rectangle Highlight、`RoundedBoxAnnotation`）の Add Box は
位置を指定せず既定位置に生成される。これを、**パレットの Add Box ボタン押下後、キャンバス上で
矩形の左上と右下を 2 回クリックして範囲を指定する** UI（パワーポイントの図形描画と同様の操作感）
に変更する。角丸み（`cornerRadiusPx`）のスクリーン空間固定という既存仕様（`CURS-L2-001`）は
変更しない。

**改訂対象**: `CURS-L1-007`（囲みの生成方法に、2クリックによる範囲指定操作を追記する L2 相当の
詳細化）。

## 6. 受入基準（Acceptance Criteria）

- [ ] ヘッダーの一番左に `[Fit]` が追加され、押下で全体表示（既存 Fit 相当）になること。
- [ ] ヘッダーの左から2番目に `[P]` が追加され、押下でパレットの表示/最小化が切り替わること。
      パレット側 `[-]` で最小化した場合も `[P]` が連動して不活性表示（色変化）になり、逆方向の
      操作でも双方が同期すること。
- [ ] `[SS]` が、現在の表示域（ビューポート）の画像をファイルダウンロードではなく
      **クリップボードへコピー**すること（`navigator.clipboard.write` 経由）。
- [ ] 画面右上のヘッダー末尾要素の左に `[en]` `[jp]` が追加され、切替により `[AI]` と `[?]` の
      表示文言が日英で切り替わること。`[AI]` のコピー結果は、プロンプト部分のみが選択言語に応じて
      和訳/英語表示され、JSON スキーマ部分は言語に依らず変化しないこと。
- [ ] パレットのカーソルボタンの右にイナズマ線トグルが追加され、新規文書での**既定が非表示**である
      こと。
- [ ] パレットに `[Ao]`（Overlap）/`[As]`（Separate）トグルが追加され、既定が `[Ao]` であり、
      切替で `viewState.planActualStyle` の描画（PLAN-L1-005）が切り替わること。
- [ ] パレットに担当者名の表示/非表示トグルが追加され、切替でアイテム上の担当者名表示が
      即時に切り替わること（レイアウト規則は CR-004 Part 5 に従う。CR-004 の所在確認後に本項目を
      最終確認する）。
- [ ] パレットの Add Box ボタン押下後、キャンバス上で左上→右下の2クリックにより丸角囲み
      （`RoundedBoxAnnotation`）の範囲が指定され、角丸みがスクリーン空間で一定（CURS-L2-001）に
      保たれること。
- [ ] 単体テスト 95% 以上／結合テスト 100%（品質目標準拠）で green であること。

  代替 or エラー通知）は実装フェーズで確定する。
