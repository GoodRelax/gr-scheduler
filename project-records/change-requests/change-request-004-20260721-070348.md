## 1. 背景・目的

CR-001〜CR-003 で予実モデル・ヘッダー配置・ラベル位置・依存線自動配線の仕様を確定してきたが、
本セッションでユーザーから実際の描画結果（マルチバー積み順・マイルストーンの見た目・左ペインの
分類ツリー・プロパティパネルのラベル配置・担当者名表示・アイコン体系）について 6 件の追加変更
要求が発生した。いずれも「表示・レイアウト・アイコン」という単一のテーマに属するため、CR-004
として 1 パッケージにまとめる。

1. マルチバー・レイアウトエンジンのサブレーン積み順を反転し、依存線の入線方向（IM8 で実装済みの
   上下端入線）と視覚的に整合させる（Part 1）。
2. マイルストーンのアイコン寸法とラベルフォントの基準を見直す（Part 2）。
3. フォント拡大時に左ペインのセクションツリーで発生する重なりを解消する（Part 3）。
4. プロパティパネルの一部フィールドのラベル配置を他フィールドと統一する（Part 4）。
5. 担当者名の表示レイアウトを新規に規定する（Part 5）。
6. アイコン体系を刷新する：外部画像インポートの廃止、塗り潰し★から輪郭☆への変更、特殊マイル
   ストーン 7 種の新設（Part 6）。

本 CR は CR-001/CR-002/CR-003 と同様に仕様（`.sdoc`）確定を先行し、`docs/api/gr-scheduler.schema.json`
および `src/**` / `tests/**` の反映は別セッション（実装フェーズ）へ引き継ぐ。

## 2. 変更内容

### Part 1 — 下から上へ積む（サブレーン積み順の反転）

同一行/セクション内で時間的に重なるアイテムのサブレーン割当順を、現行の上→下から**下→上へ
反転**する。狙いは、最上段にマイルストーンを配置し、下側のタスク群から上へ依存線が抜けていく
構図を作ることである。

- 対象: マルチバー・レイアウトエンジン（`ARCH-C-011` が実装する ITEM-L1-001「1 行への複数
  アイテム配置」の重なり回避アルゴリズム、および ALIGN-L1-001「上下左右揃え」の上下揃えの
  基準）。
- 積み順反転後も、既存の上下左右揃え（ALIGN-L1-001）・ラベルはみ出し衝突回避の縦オフセット
  （ALIGN-L2-003）の意図は維持する。積み順という「割当順序の起点」のみを反転させる。
- **対象アイテム: タスクバーとマイルストーンの両方に適用する（ユーザー確認済み 2026-07-21）。**
  マイルストーンも、アイコン＋略称（abbreviation）が同一行/セクション内の他アイテムと衝突・重なる
  場合は、タスクバーと同一規則で下→上へ積む。
- Rationale: 最上段=マイルストーンという構図は、IM8 で実装済みの依存線の上下端入線
  （DEP-L1-003「後続の上端/下端から侵入」、DEP-L2-001）と組み合わせたとき、下のタスク群から
  上のマイルストーンへ依存線が視覚的に「立ち上る」構図を作り、一枚絵としての可読性を高める。

### Part 2 — マイルストーンのサイズ/フォント

- マイルストーンアイコンの高さを、タスクバーの高さ比で **+15% 大きく**する。
- マイルストーンのラベル（略称）フォントサイズは、**マイルストーンアイコンの寸法を基準に決定**
  する。これはタスクがタスクバーの高さを基準にフォントサイズを決めるのと同様の考え方である。
- 対象: ITEM-L1-004「既定図形セット」の描画寸法、ITEM-L2-003「マイルストーンラベルの表示位置
  （アイコン右）」のフォント算出基準。
- Rationale: マイルストーンは点状の記号であるため、タスクバーと同一の高さでは相対的に小さく
  見えて視認性が劣る。+15% のサイズアップとアイコン基準のフォント算出により、一枚絵上でマイル
  ストーンの存在感をタスクと釣り合わせる。

### Part 3 — フォント大時のセクション領域マージン

フォントを大きく設定すると、左ペインのセクションツリー（大分類/中分類/小分類の階層表示、
PROP-L1-002 の `major_category` / `middle_category` / `minor_category`）で、小分類の一番上の
行が中分類の行にかぶって視認性が損なわれる不具合的挙動が確認された。行の高さマージンを増やして
重なりを回避する。

- 対象: 左ペインのセクションツリー描画（レイアウトエンジン `ARCH-C-011` が算出する行・セクション
  の縦配置）。
- Rationale: フォントサイズはツールバーの全 UI テキストへ一律適用される既存仕様
  （`19-tools-watermark.sdoc` TOOL-L1-002）であり、大フォント設定時でも階層間の重なりが生じない
  よう、行高マージンをフォントサイズに連動して確保する必要がある。

### Part 4 — プロパティのラベル配置統一

プロパティパネル（PROP-L1-001「アイテムへのプロパティ設定」）において、`end_date` /
`fade_in_days` / `fade_out_days` / `actual_end` の 4 フィールドのみラベルが**値入力部の上**に
配置されており、他のフィールド（`start_date` 等）と配置が不統一である。これを他プロパティと
同様に**値入力部の左・右詰め**（ラベル右詰め＋値入力部左揃え、他フィールドと同一レイアウト）へ
統一する。

- 対象: プロパティパネルのフィールドレイアウト実装。データモデル・フィールド定義自体
  （PROP-L1-002）は変更しない（見た目のレイアウトのみの変更）。
- 補足（仕様の既存ギャップ）: `fade_in_days` / `fade_out_days` はスキーマ（`fadeInDays` /
  `fadeOutDays`）およびドメインモデル（`docs/spec/_assets/domain-model-class.md`）には存在する
  が、PROP-L1-002 のプロパティ項目セット表には未列挙である。本 CR はレイアウト配置のみを扱い、
  当該表への追記要否は別途 srs-writer が判断する事項として本 CR のスコープ外に置く。

### Part 5 — 担当者名の表示レイアウト

担当者名（PROP-L1-002 の `assignee`）を表示する場合、**タスク/マイルストーンの左**に、**依存線の
入口（→）の水平部分と被らない**ように配置し、**右詰め**で表示する。

例（等幅、右詰め）:

```
          鈴木 →[納品]
      早乙女 →[監督]
        鈴木 →[実行]
  田中 →[計画]
```

- 対象: アイテムラベル描画（ITEM-L1-009「略称のアイテム内部/付近への表示」の周辺情報表示）と、
  依存線の左端入線（DEP-L1-002 の `middle_left` アンカー、DEP-L1-003 の左辺入規則）との非干渉
  レイアウト。
- 担当者名の表示/非表示を切り替えるトグル UI 自体は **CR-006**（パレット関連 CR）で扱う。本
  Part は表示するときのレイアウト規則のみを定める。

### Part 6 — アイコン体系刷新

**(a) アイコンのインポート（SVG/PNG 取込）を廃止する。**

- ITEM-L1-007「外部アイコン画像のインポート」、ITEM-L1-008「インポート形式は SVG および PNG に
  限定」、ITEM-L2-001「インポート画像の検証とサニタイズ」を撤廃対象とする。
- `ARCH-C-026`（インポートサニタイザ）のうち画像（SVG/PNG）経路を撤去する。JSON/MSPDI XML の
  検証・サニタイズ（IO-L1-006）は本 CR の対象外であり維持する。
- スキーマの `importedAssetId` フィールドを削除する。既存の `importedAssetId` 参照は未公開の
  ため互換維持は不要（下記 §4）。
- Rationale: 画像インポート経路（SVG サニタイズ・PNG 検証）はセキュリティ上の攻撃面であり、
  使用実績がないまま保持するコストに見合わない。撤去によりインポートサニタイザの責務が
  JSON/MSPDI の検証に純化され、セキュリティレビュー対象も縮小する。

**(b) 塗り潰し★から輪郭☆へ変更する。**

- ITEM-L1-004「既定図形セット」のマイルストーン `star` 図形を、現行の塗り潰し星から**輪郭
  （☆）**表示に変更する。塗り色は従来どおり ITEM-L1-006 / PROP-L1-002 の `fill_color` プロパティ
  で指定する（図形の既定描画が塗り潰しか輪郭かという初期状態の変更であり、色指定の仕組み自体は
  変更しない）。

**(c) 特殊マイルストーン 7 種を追加する。**

パレットの `[...]`（既定 7 図形に収まらない追加候補の展開 UI）から選択可能な特殊マイルストーン
図形として、以下 7 種を ITEM-L1-004 の既定図形セットに追加する。

| No | 図形 | 意味 |
|----|------|------|
| 1 | ファイル（右下が少し折れた紙） | 図面/資料提出期限 |
| 2 | 立体の箱 | ECU 等の物体（ハードウェア成果物） |
| 3 | フロッピー | SW リリース |
| 4 | 円筒 | サーバーへ SW デプロイ |
| 5 | シンプルな上半身の輪郭 | 偉い人に報告 |
| 6 | シンプルな笑顔 | 楽しいイベント |
| 7 | ビアタンブラー | 飲み会 |

- 対象: スキーマの `milestoneShape` enum 拡張（実装セッションで `docs/api/gr-scheduler.schema.json`
  へ 7 種を追加）。`taskShape` enum は本 Part の対象外（変更なし）。
- Rationale: すり合わせ開発の日程表では、図面提出・SW リリース・デプロイ・報告・イベント等の
  「意味のある特殊マイルストーン」を絵文字（ITEM-L1-005）に頼らず既定図形として持つことで、
  アイコンインポート廃止（Part 6a）後も表現力を維持する。
