## 1. 背景・目的

ヘルプ画面（ヘッダー `[?]` から開くモーダル、実装 `src/adapters/ui/help-modal.ts`）は、`buildHelpModel()`
が 7 セクション・約 40 項目のカタログをフラットな配列で保持し、`.grsch-help-columns` に
`column-count: 3` の CSS 段組みで流し込んでいる。現行スタイルは次の通り（読取専用で確認済み）:

- `.grsch-help-dialog { width: 85vw; max-height: 92vh; overflow: auto; font-size: 13px; }`
- `.grsch-help-columns { column-count: 3; column-gap: 24px; }`
- レスポンシブ縮退: `@media (max-width: 900px)` で 2 段組み、`@media (max-width: 620px)` で 1 段組み

すなわち現行は `overflow: auto` により**縦スクロールを前提**とした実装であり、`max-height: 92vh` を
超える内容は画面内に収まらずスクロールが発生する。またコメント（Help モーダルの JSDoc）に「English
only（ヘルプ表示は英語固定、製品方針）」とある通り現行は英語表示のみだが、CR-006 Part 4 により将来
`[en]`/`[jp]` トグルでヘルプの表示文言を日英切替する方針が既に確定している。日本語は英語より文字幅が
広くなりやすく、また CR-010 でヘルプ到達動線に `Download GR Scheduler` ボタンが追加される予定もあり、
今後ヘルプの内容量・表示言語が変化してもワンスクリーンに収まり続けるための**レイアウト制約**を本
CR で確定する。

---

## 2. 変更内容

### Part 1 — ワンスクリーン収まり（スクロール禁止）

ヘルプ画面は、**日本語表示・英語表示のどちらであっても**、1 画面内に**縦スクロールなしで**全内容
（全 7 セクション・全項目）が収まること。現行の `overflow: auto` によるスクロール許容（実質的な
「はみ出したらスクロールで見る」設計）を廃止し、はみ出しが発生しないレイアウトへ変更する。

### Part 2 — レイアウト方針（3 段組み維持）

現行の **3 段組み（`column-count: 3`）は維持**する。段組みを崩して 1 列/2 列にする、または項目を
タブ切替・アコーディオン等の別 UI パラダイムへ変更することは行わない。

### Part 3 — 幅の活用（利用可能領域まで拡大）

現行 `width: 85vw` は幅に余裕があるにもかかわらず使い切れていない。ヘルプ表示の**幅を、利用可能な
画面領域（ビューポート）まで、現行よりさらに広げる**（`85vw` を拡大する、または余白/パディングを
縮小する等により、実質的な表示幅を最大化する）。

### Part 4 — フォント調整の順序（幅拡大を優先、フォント微縮は最終手段）

ワンスクリーンに収める際の対応順序を次の通り固定する。

1. まず **Part 3 の幅拡大**で 1 画面に収まるようにする。
2. 幅拡大でも収まらない場合に**限り**、ヘルプ本文（`.grsch-help-entry` 等の項目テキスト、現行
   `font-size: 13px` 相当）を**少しだけ**小さくして収める。
3. 段組みを崩す（Part 2 に反する 1 列/2 列化）ことや、縦スクロールを導入する（Part 1 に反する）
   ことは、フォント微縮より**優先度が低い**（＝最後の手段としても採らない）。

注記: 現行のレスポンシブ縮退（`max-width: 900px` で 2 段組み、`max-width: 620px` で 1 段組み）は
Part 2「3 段組み維持」と字義上は矛盾しうる。ただし本 CR が主眼とするのは通常の実行時ウィンドウ幅
（デスクトップ利用を前提とする本製品の主要ユースケース）での挙動であり、狭幅ブレークポイントの
扱い（廃止するか、3 段組みを最小幅で維持する方向へ調整するか）は実装フェーズで確定する（下記 §7
未決事項）。

### 注記 — CR-006 / CR-010 との整合

CR-006 Part 4（`[en]`/`[jp]` 言語トグル、ヘルプの表示文言を日英切替）および CR-010（ヘルプ到達動線
への `Download GR Scheduler` ボタン追加）により、ヘルプ画面に表示される言語・要素は将来増減する。
これらを Help に反映する際も、本 CR のワンスクリーン制約（Part 1〜4）を満たすこと。すなわち、
日本語化やボタン追加によって 1 画面に収まらなくなる場合も、対応順序は Part 4 の通り「幅拡大 →
（それでも収まらなければ）フォント微縮」であり、スクロール導入や段組み崩しでは解決しない。

## 6. 受入基準（Acceptance Criteria）

- [ ] 日本語表示・英語表示のどちらでも、ヘルプ画面の全内容（全セクション・全項目）が縦スクロール
      なしで 1 画面に収まること。
- [ ] ヘルプ画面のレイアウトが 3 段組み（3 カラム）で維持されること（1 列/2 列化や段組み崩しを
      行わないこと）。
- [ ] ヘルプ画面の表示幅が、利用可能な画面領域（ビューポート）まで現行より拡大されること。
- [ ] フォントサイズの縮小は、幅拡大のみでは 1 画面に収まらない場合に**限り**適用され、かつ
      「少しだけ」の縮小に留まること（可読性を大きく損なう縮小は行わない）。
- [ ] CR-006（`[en]`/`[jp]` 言語トグル）適用後・CR-010（`Download GR Scheduler` ボタン）適用後の
      ヘルプ内容量でも、上記ワンスクリーン制約（スクロール無し・3 段組み維持・幅優先・フォント
      微縮は最終手段）が満たされること。

## 7. 決定事項・未決事項

**決定済み（本セッションでユーザー確定、2026-07-21）:**
- ヘルプ画面は日英どちらの表示でもスクロールなしで 1 画面に収まること。
- 3 段組みレイアウトは維持する。
- 幅は利用可能領域まで拡大する（現行 `85vw` からの拡大）。
- 収まらない場合の対応順序は「幅拡大 → （それでも収まらない場合のみ）フォント微縮」であり、
  段組み崩しやスクロール導入より優先する。

**未決事項（実装フェーズで確定）:**
- 現行のレスポンシブ縮退（`max-width: 900px` で 2 段組み、`max-width: 620px` で 1 段組みへの縮退）
  を、3 段組み維持の方針のもとでどう扱うか（廃止する／最小幅を再定義して 3 段組みを可能な限り
  維持する等）。
- 幅拡大の具体的な数値（`85vw` から何 vw/px 相当に広げるか、パディング/ガター調整を含むか）。
- フォント微縮が必要になった場合の具体的な下限値（可読性を損なわない最小フォントサイズ）。
- CR-006 の `[en]`/`[jp]` トグル実装後の日本語ヘルプ本文の文字数・レイアウトを踏まえた最終確認。

`tests/**` の実装は一体で反映する方針を踏襲する）により、本フェーズでは以下とする。

- **本フェーズで作成するのは本 CR 票のみ**とし、`docs/spec/**`・`docs/api/**`・`src/**`・
  `tests/**` は一切変更しない。
- **実装フェーズで反映する**: `src/adapters/ui/help-modal.ts`（`ensureHelpModalStylesheet` の CSS
  調整）、および対応するテスト。実装は一体で行い、緑（品質目標）を保つ。
- `docs/spec/19-tools-watermark.sdoc` への要求追記要否（`TOOL-L1-008` への詳細化として明記するか、
  実装コメントに留めるか）は実装フェーズで判断する。
