## 1. 背景・目的

本製品は GitHub Pages（Actions によるビルド・公開）で公開する単一 `.html` アプリである。公開済み
Web アプリ自身から配布用の単一 HTML を取得できない場合、ユーザーが手元にアプリ本体を保存・再配布
するには GitHub Releases でのタグ打ち運用（ビルド成果物の添付）が必要になる。しかし本プロジェクトは
ほぼ全自動開発で高頻度に変更されるため、タグ/Release を都度打つ運用は煩雑である。

本セッションでの検討の結論（★0）として、**公開 Web アプリ自身にワンクリックで単一 HTML をダウン
ロードできるボタンを設ける**ことで、タグ/Release 運用を必須ではなくす方針を確定した。これにより、
閲覧者は「今開いている公開ページ」からそのままアプリ本体（単一 HTML）を保存・再配布できる。

---

## 2. 変更内容

### Part 1 — [Download GR Scheduler] ボタン（ヘッダーの [?] Help から到達）

ヘッダー最右要素である **[?]（Help、`TOOL-L1-008` の第15要素）から到達できる位置**（Help
ポップアップ/メニュー内）に、**「Download GR Scheduler」ボタン**を新設する。ヘッダーの並び順
（`GR Scheduler +(c)` 〜 `?`）自体に新規トップレベル要素を追加するものではなく、既存の `?` の
配下（Help 到達動線）に機能を追加するものである（CR-003 が確定したヘッダー15要素の並びは変更
しない）。

- **用途**: 閲覧者が、公開 Web アプリ（GitHub Pages 上の単一 HTML）を開いた状態のまま、単一
  HTML アプリ本体をダウンロードフォルダへ保存し、手元での再配布・オフライン利用を可能にする。
- 押下すると、後述 Part 2 の方式で `gr-scheduler.html` がダウンロードされる。

### Part 2 — ダウンロード対象は「クリーンなアプリ本体」（現在編集中データを含まない）

ダウンロードされる HTML は、**ユーザーが現在編集中のデータ（アイテム・依存線・プロパティ等の
文書状態）を一切含まない、配信された状態そのままの素のアプリ**とする。

- **実装方針（推奨）**: `fetch(location.href)` により、自分自身が配信された元の HTML テキストを
  再取得し、それを `Blob` 化して `gr-scheduler.html` というファイル名でダウンロードする（File API、
  `IO-L1-004` に類する明示的なローカル保存の一種）。
- **明示的に採らない方式**: 現在の DOM（`document.documentElement.outerHTML`）をそのまま書き出す
  方式は、ユーザーが編集中の文書状態（localStorage 自動保存の対象と同様の、その場の編集内容）が
  DOM に混入し得るため**採らない**。ダウンロード物は常に「配信されたそのままのアプリ」であること
  を優先する。
- ファイル名は固定で `gr-scheduler.html` とする。

### Part 3 — 配布方法における位置づけ（Release/タグ運用の必須解消、オフライン時の扱い）

- 本ボタンの新設により、GitHub Releases/タグ運用は**配布手段として必須ではなくなる**（★0の
  結論）。公開ページ自体が常に最新のダウンロード元となるため、タグを打たずとも最新版を配布できる。
- 将来、版管理付き配布（特定バージョンの固定配布、変更履歴付きリリースノート等）が必要になった
  場合は、別途 Release ワークフローを追加してよい。これは**本 CR の範囲外**とする。
- **オフライン時の注記**: 本アプリを `file://` で（ローカルファイルとして直接）開いている場合、
  `fetch(location.href)` は CORS/スキーム制約により失敗し得る。この場合でも、ユーザーは既に
  ローカルにファイル本体を保有している（そもそも `file://` で開けているのはファイルを既に持って
  いるため）ため、実害はないものとして扱う。失敗時の UI 上の扱い（無処理・エラートースト等の
  具体的な通知方法）は実装フェーズで確定する。

## 6. 受入基準（Acceptance Criteria）

- [ ] ヘッダーの `[?]`（Help）から「Download GR Scheduler」ボタンに到達できること。
- [ ] 「Download GR Scheduler」押下で `gr-scheduler.html` がダウンロードされること。
- [ ] ダウンロードされる HTML が、ユーザーが編集中のデータ（アイテム・依存線・プロパティ等）を
      含まない、配信された素のアプリであり、ダウンロードしたファイルを開くと単体で正しく動作する
      こと（`document.documentElement.outerHTML` 由来の混入データを含まないこと）。
- [ ] オフライン（`file://` で開いた場合等）で `fetch` が失敗しても、実害のない扱い（既にファイルを
      保有しているため機能不全とみなさない）となること。

  `docs/spec/20-io-interop.sdoc`（ダウンロード出力種別の追記要否判断）。
