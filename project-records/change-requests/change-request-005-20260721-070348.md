## 1. 背景・目的

`docs/spec/19-tools-watermark.sdoc` の TOOL-L1-002 は、フォントサイズ設定を「大/中/小」の
3 段階として定義し、コード上は `fontScale: 'S' | 'M' | 'L'` としてヘッダーのボタン
`[A-][A][A+]`（`src/app/main.ts` の `fontGlyphs`）で切り替える実装になっている。本セッションで
ユーザーから、このアイコン表記と適用範囲・実際の追従状況について、以下 4 点の改善要求が確定した。

1. **表記の誤解を招く**: `[A-][A][A+]` は「拡大縮小の演算子」に見えるため、離散 3 段階を表す
   `[S][M][L]` へ改称する（値 `fontScale: 'S' | 'M' | 'L'` 自体は変更しない。表示アイコンのみ
   変更）。
2. **適用対象が仕様上あいまい**: 現行 TOOL-L1-002 は「スケジュールラベル・プロパティパネル・
   ヘッダーを含むすべての UI テキスト」への一律適用と定めるが、本 CR で対象を
   **セクション（分類名）・プロパティパネル・アイテムのコメント** に絞り込み、
   **ヘッダーとパレットは対象外（現状維持）** と確定する。これは現行 TOOL-L1-002 の文言
   （ヘッダーを含む一律適用）を上書き・特定化する変更である。
3. **プロパティパネルが収まらない場合がある**: `src/adapters/ui/property-panel.ts` は現状スクロール
   なしで収まるようチューニング済みだが、L（大）選択時に全項目が収まるかは要調整。
4. **コメントのフォントが追従しない**: `src/adapters/render/layers/comment-layer.ts` はコメント
   本文の `font-size` を `'12'` に固定しており、フォントスケールに応じて変化しない
   （CURS-L2-002 の位置モデルはこの点を規定していない）。これを、**小分類のセクション名と同じ
   フォントサイズ** に追従させる。

**要確認（ユーザー確認事項）**: Part 2 で列挙した「セクション（分類名）/ プロパティパネル /
アイテムのコメント」以外にフォントスケール対象とすべき UI 要素があるかは未精査。本 CR 承認後、
実装セッション開始前に対象漏れがないか再確認すること。

## 2. 変更内容

### Part 1 — フォント操作アイコンの表記変更

- ヘッダーのフォントサイズ切替ボタンの表示を、現行 `[A-][A][A+]` から **`[S][M][L]`** へ改称する。
- 内部値 `fontScale: 'S' | 'M' | 'L'`（`docs/api/gr-scheduler.schema.json` の
  `fontScale` enum、`src/app/font-scale.ts` の `UI_FONT_PX_BY_SCALE`）は変更しない。表記（アイコン
  文字列）のみの変更であり、データ形式・保存 JSON への影響はない。
- 対象要求: TOOL-L1-002（現行タイトル「全体フォントサイズの切替 (大/中/小)」の STATEMENT/
  VERIFICATION 文中に `[S][M][L]` 表記を追記）。TOOL-L1-008（ヘッダーのボタン配置順序表）の
  該当行の表記説明も整合させる。

### Part 2 — フォントスケール対象オブジェクトの明確化

- `[S][M][L]` で **フォントが変わる対象** を次の 3 種に確定する。
  1. **セクション（分類名）**: 左ペインに表示される大分類・中分類・小分類の名称
     （SECT-L1-006 分類名の左ペイン表示、SECT-L2-001 階層のインデント表現）。
  2. **プロパティパネル**: アイテムのプロパティ編集 UI 全体（PROP-L1-001 プロパティ設定、
     PROP-L1-002 27 項目のラベル・値表示）。
  3. **アイテムのコメント**: キャンバス上のコメント本文テキスト（CURS-L1-005 コメント配置、
     CURS-L1-006 引出し四角/折れ線、CURS-L2-002 位置モデル）。
- **ヘッダーとパレットのフォントは現状維持とし、`[S][M][L]` のスケール対象外とする。**
  - ヘッダー: TOOL-L1-008 が規定するボタン群（`GR Scheduler +(c)` 〜 `?`）およびタイトル表示領域。
  - パレット: TOOL-L1-001 フローティングツールパレット（形状/アイコン選択パレット）。
- 上記により、現行 TOOL-L1-002 の「ヘッダーを含むすべての UI テキストへ、同一の倍率で一律かつ
  即時に適用する」という文言は、本 CR 承認後に「セクション（分類名）・プロパティパネル・
  アイテムのコメントへ適用し、ヘッダー・パレットは対象外とする」へ改訂する。
- **要確認**: 上記 3 対象以外（例: ダイアログ、Undo/Redo 履歴表示、透かし文字列等）に
  フォントスケールを適用すべきかは本 CR の範囲外とし、別途ユーザー確認を要する。

### Part 3 — プロパティパネルの収まり

- `[S]` / `[M]` / `[L]` のいずれのフォントサイズを選択した場合でも、プロパティパネル
  （`src/adapters/ui/property-panel.ts`）が高さ方向に収まり、**スクロールせずに全項目が見える**
  よう、フォントサイズおよび行高（項目間の余白）を調整する。
- 現状 `property-panel.ts` は `overflowY: 'auto'` を安全策として残しつつ、通常のウィンドウ高さで
  スクロール不要となるようチューニング済み（コメント: 「Smaller than before so all rows fit without
  scrolling」）だが、この調整は特定のフォントサイズ（実装時点の既定 M 相当）を前提にしたもので
  あり、`[L]` 選択時に 27 項目（PROP-L1-002）すべてが収まるかは未検証。本 CR は `[S][M][L]`
  いずれでも収まることを新たな受入基準とする。
- 対象要求: PROP-L1-001、PROP-L1-002（27 項目セット）。TOOL-L1-002 の適用先としての
  プロパティパネルの記述と整合させる。

### Part 4 — コメントのフォント追従

- 現状、コメント本文の `font-size` は `src/adapters/render/layers/comment-layer.ts` にて
  固定値 `'12'` で描画されており、`[S][M][L]` を切り替えてもコメントのフォントサイズは
  **変化しない**（defect 相当の未追従）。
- 本 CR により、コメント本文のフォントサイズが `[S][M][L]` に応じて **変わるようにする**。
- サイズは **小分類のセクション名と同じフォントサイズ** とする（Part 2 で確定したセクション
  （分類名）用のフォントサイズ値をコメントにも適用し、両者を同一の値にそろえる。分類の階層
  （大分類/中分類/小分類）のうち、最下位の「小分類」の表示サイズを基準値とする）。
- 対象要求: CURS-L1-005（コメント配置）、CURS-L1-006（引出し四角/折れ線）、CURS-L2-002
  （位置モデル。フォントサイズはこのモデルの新規属性ではなく、グローバルなフォントスケール
  設定への追従として扱う）。SECT-L1-006/SECT-L2-001（基準となるセクション名フォント）。

## 6. 受入基準（Acceptance Criteria）

- [ ] ヘッダーのフォントサイズ切替ボタンが `[S]` `[M]` `[L]` の表記で表示されること
      （`[A-][A][A+]` ではないこと）。内部値 `fontScale` は `'S' | 'M' | 'L'` のまま変更されて
      いないこと。
- [ ] `[S][M][L]` の切り替えにより、セクション（分類名: 大分類・中分類・小分類）、プロパティ
      パネルの表示、アイテムのコメント本文のフォントサイズが変化すること。
- [ ] `[S][M][L]` の切り替えによって、ヘッダー（`GR Scheduler +(c)` 〜 `?` の各ボタン・タイトル
      表示）およびパレット（フローティングツールパレット）のフォントサイズが変化しないこと
      （現状維持）。
- [ ] `[S]` `[M]` `[L]` いずれの設定でも、プロパティパネルが通常のウィンドウ高さでスクロール
      せずに全 27 項目（PROP-L1-002）を表示できること。
- [ ] コメント本文のフォントサイズが、同時点の小分類セクション名のフォントサイズと一致すること
      （`[S][M][L]` いずれの設定でも一致すること）。
- [ ] 単体テスト 95% 以上／結合テスト 100%（品質目標準拠）で green であること。

## 7. 決定事項・未決事項

**決定済み（本セッションでユーザー確定、2026-07-21）:**
- フォントサイズ切替アイコンを `[A-][A][A+]` から `[S][M][L]` へ改称する（値は不変）。
- フォントスケール `[S][M][L]` の適用対象を「セクション（分類名）・プロパティパネル・アイテムの
  コメント」の 3 種に確定する。
- ヘッダーとパレットのフォントは現状維持とし、フォントスケールの対象外とする。
- プロパティパネルは `[S][M][L]` いずれでもスクロールなしで全項目が収まるよう調整する。
- コメントのフォントサイズは `[S][M][L]` に追従させ、値は小分類セクション名のフォントサイズと
  同一とする。

**未決事項（実装セッション開始前に要確認）:**
- Part 2 で列挙した 3 対象（セクション名/プロパティパネル/コメント）以外に、フォントスケール
  対象とすべき UI 要素（ダイアログ、Undo/Redo 表示、透かし文字列等）があるかどうか。本 CR では
  「要確認」として未決のまま記録し、実装セッション開始前にユーザーへ再確認する。
- プロパティパネルの収まり調整が、既存の 27 項目セット（PROP-L1-002）へ将来項目が追加された
  場合にも維持可能かどうかの方針（本 CR の範囲外、将来 CR で扱う）。
