## 1. 背景・目的

CR-002 でベースライン可視トグル（Base V/I）をヘッダーへ追加することが決まったが、既存のツール
バー（`19-tools-watermark.sdoc` の TOOL-L1-001/002/004/005 系）は配置順が仕様化されておらず、
ボタン追加のたびに場当たり的に混雑する懸念があった。あわせて本セッションで次の 2 点も UI/レイア
ウトの改善要求として確定した。

1. **ヘッダー/ツールバーの再整理**: SS（表示域スクリーンショット）・Load・Save・4 種テーマ・
   ベースライン可視トグル・Undo/Redo・AI・Help を含む全ボタンの並び順と各ボタンの意味（言霊命名、
   user-order item60）を明文化する。
2. **ラベル位置の原則**: タスクラベルの既定位置（バー内左揃え）と、はみ出し時の同一セクション内
   衝突回避（マルチバー・レイアウトエンジンの責務）、マイルストーンラベルの位置（アイコン右）を
   明文化する。
3. **依存線の自動配線（自動のみ）**: DEP-L1-002 の 9 点アンカーは維持しつつ、当面は手動アンカー
   選択を用いず、右出/左入・直交・決定的な自動配線規則へ簡素化する。

本 CR は上記を Part 1〜3 として 1 パッケージにまとめ、CR-001/CR-002 と同様に仕様（`.sdoc`）確定
を先行し、`gr-scheduler.schema.next.json` / `src/**` の反映は別セッション（実装フェーズ）へ引き
継ぐ。

## 2. 変更内容

### Part 1 — ヘッダー/ツールバー再整理

ヘッダーが混雑した（CR-002 のベースライン可視トグル追加等）ため、左→右で次の順に再配置する。

`GR Scheduler +(c)` → タイトル（プロジェクト名） → SS → Load → Save → Light → Dark → Mono L →
Mono D → Base V → Base I → Undo → Redo → AI → ?（Help）

- **SS** = 表示域（現在のビューポート）のスクリーンショットを PNG 出力。
- **Load** = JSON, XML の取込。JSON の Load 時オプションで「ベースライン扱い」指定が可能
  （CR-002 §2 Part 3 連動）。
- **Save** = JSON, XML, SVG, PNG の出力。SVG/PNG は全キャンバスを Fix した全体出力（SS の表示域
  スクショとは役割が別）。
- **Light / Dark / Mono L / Mono D** = テーマ選択（4 種: ライト/ダーク/モノクロ・ライト/モノクロ・
  ダーク）。Mono は単色テーマ（印刷/色覚配慮）。
- **Base V / Base I** = ベースライン参照層の可視トグル（Visible/Invisible。CR-002 のベースライン
  層の表示切替）。
- **Undo / Redo**。
- **AI** = AI 向けプロンプト + JSON スキーマをクリップボードへコピー（既存実装。ボタン実体は定義
  済み）。
- **?** = ヘルプ。

注記: 既存 TOOL-L1-001（フローティングパレット）・TOOL-L1-002（フォントサイズ、ヘッダーを含む
全 UI テキストへ一律適用）・TOOL-L1-004（Undo/Redo）・TOOL-L1-005（ショートカット、Ctrl+S 保存）
との整合を保つ。SS/Load/Save の出力形式は IO-L1-001（JSON）・IO-L1-002（MSPDI XML）・IO-L1-003
（SVG）と対応し、Import 時の検証は IO-L1-006（外部入力の厳格な検証・サニタイズ）に従う。ボタン
名は意味ある実体を仕様に明記する（言霊命名、user-order item60）。

### Part 2 — ラベル位置の原則

- タスクのラベルは既定でタスクバー内・左揃え。`labelPosition` の値に **`inner-left`**（バー内左
  揃え）を追加する。既存の `left`（バー外・左）とは別概念として峻別する（ITEM-L1-009: 略称のアイ
  テム内部/付近への表示、ITEM-L1-010: 略称表示位置のドラッグ移動 が主管する既存モデルへの enum
  追加）。
- タスクのラベルのはみ出しは許容する。ただし、はみ出しにより同一セクション内の他タスクと表示が
  重なる場合は、当該タスクを同一セクション内で高さを変えて（縦オフセットして）重なりを回避する
  （マルチバー・レイアウトエンジンの衝突回避。ALIGN-L1-001 の上下左右揃えおよび ALIGN-L2-001 の
  整列制約ソルバの枠組みに、ラベル衝突を追加の制約として組み込む）。
- マイルストーンのラベルはマイルストーンアイコンの右に置く。

注記: `labelPosition` の enum に `inner-left` を追加する差分は次期スキーマ
（`gr-scheduler.schema.next.json`）と `40-data-format.sdoc` に反映する（実装/スキーマ差分は実装
フェーズ）。

### Part 3 — 依存線の自動配線（自動のみ）

- 依存線は当面「自動配線のみ」とする（手動 9 アンカー選択は今回使用しない。DEP-L1-002 の 9 アン
  カー仕様のうち手動選択面を簡素化/supersede。9 点アンカー自体の記述（DEP-L1-002 の座標定義）は
  保持しつつ、配線は下記規則で決定的に行う）。
- すべての依存線は先行アイテムの「右端から出て」後続アイテムの「左端へ入る」（□→ … →□）。直交
  （矩形）配線とする（DEP-L2-001 の障害物回避直交配線の枠組みを維持しつつ、始点/終点アンカーを
  右端/左端に固定する形へ specialize する）。
- 出入りの横スタブ長 = 矢印先端の 2 倍（矢印本体を含む）。矢じり自体の最小描画方針は DEP-L1-004
  に従う。
- 後続が下: 右へ出て直後に下へ折れて降下し、後続の左端へ入る（下折れは出直後）。
- 後続が上: 右へ出て可能な限り右（後続のすぐ手前）まで進み、後続の直前で縦に折れて後続の左端へ
  入る。
- 同一セクションで先行・後続が水平方向に重なり上下に積まれている場合: 折れた後の横移動は上バーと
  下バーの中間（行間のギャップ）を通し、どちらのバーも跨がない。
- 上記いずれの経路も折れ点は 0〜3 の範囲に収まる（DEP-L2-002 の制約を満たす specialize であり、
  矛盾しない）。

Rationale: 分かりやすさ優先の決定的配線。前進（後続が右）依存を主対象とし、遡り依存の詳細規則は
本 CR では前進前提（将来 CR で拡張）。

## 6. 受入基準（Acceptance Criteria）

- [ ] ヘッダーが `GR Scheduler +(c)` → タイトル → SS → Load → Save → Light → Dark → Mono L →
      Mono D → Base V → Base I → Undo → Redo → AI → ? の順に並ぶこと。
- [ ] SS が表示域のスクリーンショットを PNG で出力すること。Load が JSON/XML を取り込み、JSON
      Load 時に「ベースライン扱い」オプションを指定できること。Save が JSON/XML/SVG/PNG を出力
      し、SVG/PNG が全キャンバス Fix の全体出力であること。
- [ ] テーマが Light/Dark/Mono L/Mono D の 4 種で切り替わること。
- [ ] Base V/I でベースライン層の表示/非表示が切り替わること。
- [ ] タスクラベルが既定で `inner-left`（バー内左揃え）で描かれること。ラベルのはみ出しにより同一
      セクション内の他タスクと重なる場合、当該タスクが同一セクション内で高さを変えて重なりを回避
      すること。
- [ ] マイルストーンラベルがアイコンの右に描かれること。
- [ ] 依存線が「先行の右端から出て後続の左端へ入る」直交配線で自動描画され、横スタブ長が矢印先端
      の 2 倍であること。後続が下の場合は出直後に下折れ、後続が上の場合は後続直前で縦折れするこ
      と。水平方向に重なり上下に積まれた先行/後続間の横移動が、両バーを跨がず行間ギャップを通る
      こと。
- [ ] 上記いずれの依存線経路も折れ点が 0〜3 の範囲に収まること（DEP-L2-002 準拠）。
- [ ] 単体テスト 95% 以上／結合テスト 100%（品質目標準拠）で green であること。

## 7. 決定事項・未決事項

**決定済み（本セッションでユーザー確定、2026-07-20）:**
- ヘッダーのボタン並び順（Part 1 の 15 要素の順序）を確定。
- SS＝ビューポート PNG、Save の SVG/PNG＝全体 Fix 出力という役割分担を確定。
- テーマは Light/Dark/Mono L/Mono D の 4 種。Mono は単色（印刷/色覚配慮）。
- ラベル位置に `inner-left` を新設し、既存 `left`（バー外）と区別する。
- ラベルはみ出し時の衝突回避は「同一セクション内での高さ変更（縦オフセット）」で解決する。
- マイルストーンラベルはアイコン右固定。
- 依存線は当面「自動配線のみ」とし、手動 9 アンカー選択機能は使用しない（9 点アンカーの座標定義
  自体は DEP-L1-002 に残す）。
- 依存線の配線規則（右出/左入・直交・2×スタブ・上下折れ・行間ギャップ）を確定。前進依存
  （後続が右）を主対象とする。

**未決事項（将来 CR で拡張）:**
- 遡り依存（後続が先行より左に位置する場合）の詳細な配線規則。
- 手動 9 アンカー選択 UI を将来的に復活させるかどうか。
- ラベル衝突回避で「高さ変更」でも解決できない密集配置（同一セクション内の縦スペース不足）時の
  フォールバック規則。
