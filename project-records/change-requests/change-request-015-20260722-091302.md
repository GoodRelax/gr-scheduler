## 1. 背景・目的

本セッションでユーザーがヘッダーをライブ視覚レビューし、現行の並び順（CR-006 で確定した
`Fit → P → GR Scheduler +(c) → タイトル → SS → Load → Save → ...`）について、`[Fit]`・`[P]` が
ブランディング・タイトルより左（ヘッダーの最左）にあることに違和感を示し、次の並びへ改めるよう
要求した。

**`GR Scheduler` ブランディングを最左 → プロジェクトタイトル → `[Fit]` `[P]` →
（以降は現行と同順: SS / Load / Save / Light / Dark / Mono L / Mono D / Base V / Base I / Undo /
Redo / AI / ?）**

すなわち、ブランディングとタイトルという「ヘッダーの識別領域」を最左に置き、`[Fit]`・`[P]` は
その直後（3 番目・4 番目）に据える。`SS` 以降の並びは変更しない。

なお、ユーザーは当初 `[Fit][R]` と口頭で表現したが、本セッション内で確認したところ `[R]` は
`[P]`（パレットの表示/最小化トグル）の言い間違い（タイプミス）であったことをユーザー自身が確認・
訂正した。今後の作業者が「`[R]` という幻のボタン」を再導出することのないよう、本 CR 票に明記して
おく。`[R]` に対応する新規ボタンは存在せず、対象は既存の `[P]`（`TOOL-L1-008` 第 2 要素、
`TOOL-L2-006` のパレット最小化状態同期の対象）である。

## 2. 変更内容

`TOOL-L1-008`（ヘッダーのボタン配置順序と各ボタンの意味）の並び順を、次の通り改訂する。各ボタンの
意味（機能実体）自体は一切変更しない（CR-003・CR-006 で確定した機能はそのまま）。

### 改訂前（CR-006 Part 1・2 確定、本 CR が supersede する配置）

| 順 | ボタン |
|----|--------|
| 1 | Fit |
| 2 | P |
| 3 | GR Scheduler +(c) |
| 4 | タイトル |
| 5 | SS |
| 6 | Load |
| 7 | Save |
| 8〜17 | Light / Dark / Mono L / Mono D / Base V / Base I / Undo / Redo / AI / ? |

### 改訂後（本 CR で確定する配置）

| 順 | ボタン |
|----|--------|
| 1 | GR Scheduler +(c) |
| 2 | タイトル |
| 3 | Fit |
| 4 | P |
| 5 | SS |
| 6 | Load |
| 7 | Save |
| 8〜17 | Light / Dark / Mono L / Mono D / Base V / Base I / Undo / Redo / AI / ?（変更なし） |

すなわち、`GR Scheduler +(c)` ブランディングとタイトルの 2 要素を最左（1・2 番目）へ移し、
`Fit`・`P` をその直後（3・4 番目）へ繰り下げる。`SS` 以降（5 番目〜17 番目）の並びと各ボタンの
意味は CR-006/CR-003 の確定内容から変更しない。

`TOOL-L2-006`（ヘッダー `P` とパレット最小化ボタンの双方向状態同期）は、`P` の並び位置が変わっても
制約内容（双方向同期・不活性表示）自体には影響しない。同様に `TOOL-L1-002`（フォントサイズ切替の
ヘッダー非適用）等、他の関連要求も本 CR による並び順変更の影響を受けない。

## 8. 段階化の注意（仕様先行・実装後追い）

CR-001 §8 / CR-011 §8 / CR-012 §8 / CR-013 §8 / CR-014 §8 と同様の理由（`docs/api/
gr-scheduler.schema.json` は F0 で SSOT 化され `tests/document-schema-conformance.test.ts` がコード
出力と突合しているため、コードより先に schema を変更するとテストが赤化する。本 CR はスキーマ変更を
伴わないが、`src/**` / `tests/**` の実装は一体で反映する方針を踏襲する）により、本フェーズでは
以下とする。

- **本フェーズで作成するのは本 CR 票のみ**とし、`docs/spec/**`・`docs/api/**`・`src/**`・
  `tests/**` は一切変更しない。
- **実装フェーズで反映する**: `docs/spec/19-tools-watermark.sdoc`（`TOOL-L1-008` の並び順表・
  `VERIFICATION` 改訂）、`src/app/main.ts`（ヘッダー DOM 挿入順序の変更）、および対応するテスト。
  実装は一体で行い、品質ゲート（tsc/vitest/eslint/strictdoc/build）を緑に保つ。
