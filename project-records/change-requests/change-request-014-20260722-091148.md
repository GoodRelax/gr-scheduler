## 1. 背景・目的

本セッションでユーザーがフローティングパレットを視覚レビューし、次の課題を指摘した。

- パレット内の絵文字グリフ（7 種の特殊マイルストーンボタン、進捗線ボタン、コメント/透かし/枠追加
  ボタン等）が、既存の線画（アウトライン）スタイルと不揃いで浮いて見える。
- 進捗線ボタンの左にある `LINE` というグループラベルが不要。
- 現行は MILESTONE グループが TASK グループより先（左）に並んでいるが、マイルストーンのアイコン
  種類が増える（CR-004 の 7 種特殊形状に加え今後も増加が見込まれる）につれ、TASK アイコンが探し
  にくくなっている。
- アイコンのサイズ・間隔の比率が場当たり的で、意図的な比率設計がされていない。

読取専用でコードを確認したところ、次の事実が判明した。

- `src/adapters/ui/tool-palette.ts` の `SPECIAL_MILESTONE_CHOICES`（CR-004 Part 6c で追加した
  7 種の特殊マイルストーン: file 📄 / box3d 📦 / floppy 💾 / cylinder 🛢 / person 👤 / smiley 🙂 /
  beer 🍺）は、いずれも絵文字グリフを表示専用の記号として使っている。
- `src/app/main.ts` の `progressLineButton`（進捗線トグル、CR-006 Part 5）は絵文字 `⚡`、
  `commentButton`（コメント追加）は `💬`、`boxButton`（枠追加）は `▢`、`watermarkButton`（透かし）は
  `©` を用いる。これらのうち `⚡`/`💬` は明確な絵文字であり、`▢`/`©` は記号だが、いずれもユーザーが
  今回「全部（線画化対象）」に含めると明示的に確認した（下記 §7「決定済み」参照。厳密には `▢`/`©`
  は Unicode 絵文字ではなく記号文字だが、既存の線画アウトラインの視覚言語とは異なる描き方である
  ため、ユーザーの意図（統一されたアイコン体系）に沿い本 CR の対象に含める）。
- 一方、パレットの他のボタン（`▁`〔最小化〕・`▤`〔プロパティパネル〕・`⤢`〔Fit〕・`⛶`〔全画面〕・
  `⏱`〔ベンチマーク〕・`↔`〔リンクモード〕・`☰|`〔日付グリッド〕・`≡`〔分類グリッド〕・`@`
  〔担当者表示〕・`P`/`A`〔予定/実績表示〕等）は既に非絵文字の Unicode 記号であり、ユーザーからの
  変更指摘対象に含まれていない。本 CR の対象範囲は、ユーザーが明示した「7 種の特殊マイルストーン＋
  進捗線＋コメント＋透かし＋枠追加」の計 11 個のボタンに限定する。
- キャンバス側は既に線画 SVG パス（`src/adapters/render/item-geometry.ts` の
  `box3dGlyphPath`/`floppyGlyphPath`/`cylinderGlyphPath`/`personGlyphPath`/`smileyGlyphPath`/
  `beerGlyphPath`）で 7 種の特殊マイルストーンを描いている。パレットのアイコンは、この既存の描画
  ロジック（の縮小版、または同一パスビルダーの再利用）と**同じ視覚言語**で統一すべきである。
- `src/adapters/ui/tool-palette.ts` の `mountShapePicker` 内、
  `shapesRow.append(milestoneGroup, taskGroup);`（現行、MILESTONE が先＝左）を確認済み。
- `src/app/main.ts` の `makeCommandGroup('Line', [progressLineButton])`（`Line` グループラベル）を
  確認済み。

さらにユーザーは、参考として PowerPoint で作成した目標アイコンセット（紙〔角折れ〕/立体箱/フロッピー/
円柱/人物/笑顔/ビールジョッキ、各アウトライン線画を枠内に収めた画像）を提供した。本 CR はこの
参考画像に従うことを明記する。なお、ユーザーは追加で「タスクの形状もおかしいので、この後イメージを
貼る」と述べ、タスク形状（`TASK_CHOICES`: bar ▭ / arrow → / chevron » / span |—|）の再設計を予告
した。当該画像は本セッション時点で未提供のため、タスク形状の再設計は本 CR のスコープに含めず、
**未決事項**として下記 §7 に記録する。

## 2. 変更内容

### Part 1 — アイコンの線画化（全部）

パレット内の次の 11 個のボタンの絵文字/記号グリフを、既存のアウトライン線画スタイルと一貫した
線画（line-art）アイコンへ置き換える。

- 7 種の特殊マイルストーンボタン（`SPECIAL_MILESTONE_CHOICES`、現行 `📄`/`📦`/`💾`/`🛢`/`👤`/`🙂`/
  `🍺`）
- 進捗線ボタン（`progressLineButton`、現行 `⚡`）
- コメント追加ボタン（`commentButton`、現行 `💬`）
- 枠追加ボタン（`boxButton`、現行 `▢`）
- 透かしボタン（`watermarkButton`、現行 `©`）

7 種の特殊マイルストーンについては、キャンバスが既に持つ線画 SVG パス
（`src/adapters/render/item-geometry.ts` の `box3dGlyphPath` 等）と**同じ視覚言語**を用いる
こと。理想的には、パレットのアイコン描画がこれらのパスビルダーを（縮小サイズ用に調整した上で）
直接再利用し、キャンバスとパレットで形状の定義が二重化しない（SSOT）ようにする。現行これらの
関数はモジュール非公開（プライベート）のため、パレットから利用可能にするための export 化または
共有ヘルパーへの切り出しを実装フェーズで検討する。

進捗線・コメント・枠追加・透かしの 4 ボタンには、キャンバス側に対応する既存パスビルダーが無いため、
新規に線画アイコンを起こす（下記 Part 2 の参考画像に準じたアウトライン様式で、他の線画アイコンと
線幅・線端処理を揃える）。

### Part 2 — デザイン参照（ユーザー提供画像）

7 種の特殊マイルストーンの線画アイコンは、ユーザーが本セッションで提供した PowerPoint 製の参考
画像（紙〔角折れ〕/立体箱/フロッピー/円柱/人物/笑顔/ビールジョッキ、各アウトライン線画を枠内に
収めた画像）の形状に従うこと。当該画像は本 CR の設計入力として記録し、実装フェーズで参照する。

なお、ユーザーは追加で「タスクの形状もおかしいので、この後イメージを貼る」と述べており、タスク
形状（`TASK_CHOICES`）の再設計についても別途画像を提供する予定である。当該画像は本 CR 起票時点で
未提供のため、タスク形状の再設計は**本 CR のスコープに含めず**、画像提供後に別途 CR（または本 CR
の改訂）として扱う（下記 §7 未決事項）。

### Part 3 — `LINE` グループラベルの削除

進捗線ボタンの左に表示されている `LINE` というグループキャプション
（`src/app/main.ts` の `makeCommandGroup('Line', [progressLineButton])`）は不要であり削除する。
進捗線ボタン自体は残し、グループのラベル（キャプション文字列）のみを撤去する（グループとしての
DOM 構造自体を残すか、ボタンを他のグループへ統合するかは実装フェーズで判断する）。

### Part 4 — 並び順の変更（TASK を左、MILESTONE を右へ）

現行 `src/adapters/ui/tool-palette.ts` の `mountShapePicker` は
`shapesRow.append(milestoneGroup, taskGroup);` により MILESTONE グループを TASK グループより
左（先）に配置している。これを **`shapesRow.append(taskGroup, milestoneGroup);`（TASK が左、
MILESTONE が右）へ変更する**。マイルストーンのアイコン種類が今後も増加する見込みであるため、
種類が少なく安定している TASK を左（探しやすい固定位置）に、種類が増え続ける MILESTONE を右へ
置くことで、TASK アイコンの視認性を保つ。`[...]` 展開ボタン（`SPECIAL_MILESTONE_CHOICES` を表示
する `expander`）は MILESTONE グループに付随するため、MILESTONE グループと共に右側へ移動する。

### Part 5 — 比率の整え（黄金比/白銀比の検討）

パレットのアイコンサイズ・間隔（アイコンボックスのサイズ、グループ間の余白、グループ内のボタン
間隔等）を、場当たり的な現行値から、意図的な比率設計へ改める。黄金比（1:1.618）または白銀比
（1:1.414）を用いた寸法比率を検討し、アイコンボックス/グループ/間隔のいずれか（または組み合わせ）
に適用することで、視覚的に整った配置とする。具体的にどの寸法（アイコンボックスの一辺比、ボックスと
間隔の比、グループ幅の比等）へどちらの比率を適用するかは実装フェーズで確定する（下記 §7 未決
事項）。

## 7. 決定事項・未決事項

**決定済み（本セッションでユーザー確定、2026-07-22）:**
- パレット内の 7 種特殊マイルストーン＋進捗線＋コメント＋枠追加＋透かしの計 11 個のボタンを線画化
  する（「全部」の対象範囲として確定）。
- 7 種特殊マイルストーンの線画は、ユーザー提供の PowerPoint 参考画像の形状に従う。
- `LINE` グループラベルは削除する。
- シェイプピッカーの並びは TASK を左、MILESTONE を右にする。
- アイコンサイズ・間隔の比率設計に黄金比/白銀比の適用を検討する。

**未決事項（実装フェーズで確定、またはユーザー画像提供後に別途扱う）:**
- タスク形状（`TASK_CHOICES`: bar/arrow/chevron/span）の再設計。ユーザーが「この後イメージを貼る」
  と予告した参考画像が本 CR 起票時点で未提供のため、**本 CR のスコープには含めない**。画像提供後、
  別途 change-request（または本 CR の改訂）として扱う。
- 7 種特殊マイルストーンの線画パスを、キャンバス描画（`item-geometry.ts`）のパスビルダーと完全に
  共有（SSOT）するか、パレット専用に別途縮小最適化した独自パスを起こすかの実装方針。
