## 1. 背景・目的

`docs/spec/19-tools-watermark.sdoc` の TOOL-L1-007（透かし付与）配下、TOOL-L2-001（透かしの内容:
ローカル入力ユーザー名 + 生成日時）・TOOL-L2-004（既定表示: 既定オン・既定文字列 `GoodRelax`・
不透明度 0.06・日時は UTC ISO 8601 分精度）・TOOL-L2-005（透かし非表示のパスワード保護。既定
パスワード `watermark-unlock`、その SHA-256 を `DEFAULT_WATERMARK_HIDE_PASSWORD_HASH` としてコード
保持。生パスワードは保存せずハッシュのみを保持）は既に確定済みである。

現行実装（`src/app/main.ts` の `wireWatermark`）を確認したところ、UTC タイムスタンプは透かしを
「非表示→表示」に切り替えた瞬間にのみ `formatWatermarkTimestampUtc(Date.now())` で採番されており、
以後はアイテム編集を重ねても更新されない。また既定（`resolveWatermark` が未保存値を解決する経路）
では `timestamp: ''` となり、日時は必須ではなく「含める場合」の任意表示にとどまっている
（TOOL-L2-004 の文言も「日時を含める場合は」という条件文であり、必須とは明記していない）。

本セッションでユーザーから次の 3 点の変更要求が生じた。いずれもユーザー起点の requirement-change
であり、change-manager を経由して記録する。

1. **既定パスワードの変更**: 透かし非表示の既定パスワードを `watermark-unlock` から
   `GoodRelax` へ変更する（TOOL-L2-005 の定数値差し替え）。
2. **透かし内容へ UTC 時刻を常時含める**: 透かしの内容を「ローカル入力ユーザー名」のみの表示を
   許さず、**UTC 時刻を必須要素として常に含める**（TOOL-L2-001 の「日時」を必須化し、TOOL-L2-004
   の「日時を含める場合は」という条件付けを解消する）。
3. **UTC 時刻の更新タイミングの明文化**: UTC 時刻は「アイテムが変更されたタイミング（= 文書の
   JSON 値が変わったとき）」に更新し、ズーム・スクロール等の表示操作（ビューポート操作）では
   更新しない、という規則を新規に明文化する（現行実装のトグル時採番を置き換える）。

本 CR は上記を Part 1〜3 として 1 パッケージにまとめ、CR-001/002/003 と同様に仕様（`.sdoc`）確定
を先行し、コード（`schedule-model.ts` の定数・`watermark-builder.ts` の生成/更新ロジック・
`main.ts` の結線）の反映は別セッション（実装フェーズ）へ引き継ぐ。**本 CR では
`docs/spec/*.sdoc`・schema・`src/**`・`tests/**` を一切編集しない**（仕様のみを先行して合意する
ための記録票である）。

## 2. 変更内容

### Part 1 — 既定パスワードの変更（TOOL-L2-005）

透かし非表示の既定パスワードを次のとおり変更する。

- 既定パスワード（平文）: `GoodRelax`（旧: `watermark-unlock`）
- **生パスワードは保存しない**方針は既存どおり据え置く。ブラウザ側（内部モデル / 出力
  HTML・JSON）に格納するのは、Web Crypto (`crypto.subtle.digest('SHA-256', ...)`) で算出した
  `GoodRelax` の **SHA-256 16 進ハッシュ値のみ**であり、コード上の定数名は既存の
  `DEFAULT_WATERMARK_HIDE_PASSWORD_HASH`（`src/domain/model/schedule-model.ts`）を維持したまま
  値のみを新パスワードのハッシュへ差し替える。
- 注記: 本 CR は方針とパスワード平文を確定するものであり、**実際の 16 進ハッシュ文字列の算出は
  実装セッションで `crypto.subtle` を用いて行い**、`DEFAULT_WATERMARK_HIDE_PASSWORD_HASH` へ反映
  する（本票は算出済みハッシュ値を記載しない。手計算によるハッシュ値の記載は誤りを生む
  リスクがあるため避ける）。
- TOOL-L2-005 の「既定パスワードはサーバー配備時のローテーション用の記録であり、サーバー配備時
  には必ず変更すること」という運用注記は変更しない（本 CR はクライアント単体運用における既定値の
  差し替えのみ）。

### Part 2 — 透かし内容へ UTC 時刻を常時含める（TOOL-L2-001, TOOL-L2-004）

- 透かし文字列は、**現在のログインユーザー名（自己申告・ローカル入力、サーバー認証なし）に加え、
  UTC 時刻を必須要素として常に含める**。TOOL-L2-004 の「日時を含める場合は UTC の ISO 8601 形式
  （末尾 `Z`、分精度）で表示する」という条件文を、「日時は必須表示であり、UTC の ISO 8601 形式
  （末尾 `Z`、分精度、例: `2026-07-19T05:12Z`）で表示する」へ改める。
- 新規文書の既定表示（既定オン・既定文字列 `GoodRelax`）についても、UTC 時刻を必須で含む
  （現行の `resolveWatermark` 未保存経路が返す `timestamp: ''` の空文字許容を解消し、常に有効な
  UTC 時刻を持たせる）。
- 表示フォーマットおよび不透明度 0.06 は TOOL-L2-004 の既存規定を維持する。

### Part 3 — UTC 時刻の更新タイミング（新規要求。TOOL-L2-001 の運用細則）

- UTC 時刻は、**アイテム（マイルストーン/タスク/依存線/セクション等、文書の JSON 値）が変更
  されたタイミングでのみ更新**する。すなわち Undo/Redo を含む編集操作によって `viewState.watermark`
  以外の文書内容が変化した直後に、透かしの `timestamp` を現在の UTC 時刻へ再採番する。
- **ズーム・パン（スクロール）等のビューポート操作（表示状態のみの変更）では更新しない。**
  `viewState` のうち `zoomX` / `zoomY` / `scrollX` / `scrollY` 等の表示専用フィールドの変化は
  「アイテムの変更」に該当しないため、UTC 時刻の再採番トリガーとしない。
- 現行実装（`src/app/main.ts` の `wireWatermark` が透かしトグル ON 時にのみ採番する挙動）は本規則
  で置き換える。透かしの表示/非表示切替（TOOL-L2-003）自体は `viewState.watermark.enabled` の
  変更であり、それ自体は「アイテムの変更」ではないため、トグル操作単体では UTC 時刻を更新しない
  （直近のアイテム変更時刻を保持したまま表示/非表示のみ切り替わる）。

Rationale: 「いつ時点の内容の透かしか」を証跡として正確にするため、時刻は内容（アイテム）の変更に
連動させる。表示操作（ズーム/スクロール）は内容を変えないため、時刻を無用に書き換えて「常に今」に
見せかけることを避ける。

## 6. 受入基準（Acceptance Criteria）

- [ ] 透かし非表示の既定パスワードが `GoodRelax` であること（誤ったパスワードでは非表示にならず、
      正しい既定パスワード `GoodRelax` で非表示になること）。
- [ ] 生パスワード `GoodRelax` がモデル・出力（JSON/HTML）のいずれにも現れず、
      `DEFAULT_WATERMARK_HIDE_PASSWORD_HASH` として SHA-256 ハッシュ値のみが保持されること。
- [ ] 透かしの内容に、ローカル入力ユーザー名に加えて UTC 時刻（ISO 8601、末尾 `Z`、分精度）が
      常に含まれること（新規文書の既定表示を含め、日時が空になるケースがないこと）。
- [ ] アイテム（マイルストーン/タスク/依存線/セクション等）を変更すると、透かしの UTC 時刻が
      その変更直後の現在時刻へ更新されること。
- [ ] ズームまたはスクロール（パン）のみを行った場合、透かしの UTC 時刻が更新されない
      （直前のアイテム変更時点の時刻を保持したままであること）。
- [ ] 透かしの表示/非表示切替（TOOL-L2-003）単体の操作では、UTC 時刻が更新されない
      （直近のアイテム変更時刻を保持したまま表示/非表示のみが切り替わること）。
- [ ] 単体テスト 95% 以上／結合テスト 100%（品質目標準拠）で green であること。

## 7. 決定事項・未決事項

**決定済み（本セッションでユーザー確定、2026-07-21）:**
- 透かし非表示の既定パスワードを `GoodRelax` に変更する。
- 生パスワードは保存せず、SHA-256 ハッシュ値のみをブラウザ側（モデル/出力 HTML）に保持する方針を
  維持する（既存 TOOL-L2-005 方針の明文化）。
- 透かしの内容に UTC 時刻を必須要素として常に含める。
- UTC 時刻はアイテム変更（JSON 値変化）のタイミングでのみ更新し、ズーム・スクロール等の表示操作
  では更新しない。

**未決事項（実装セッションで確定）:**
- `GoodRelax` の SHA-256 16 進ハッシュ値そのもの（本票では算出しない。実装セッションで
  `crypto.subtle` により算出し `DEFAULT_WATERMARK_HIDE_PASSWORD_HASH` へ反映する）。
- 「アイテムが変更されたタイミング」を検知するための具体的な結線点（Undo/Redo コマンドスタックへの
  push 時点を採用するか、状態ストアの diff 検知を新設するか）。既存の Undo/Redo 実装
  （TOOL-L1-004）との結線方式は実装セッションで設計する。
- `docs/api/gr-scheduler.schema.json` の `watermark.timestamp` の型定義自体に変更が必要かどうかの
  最終確認（現時点では型変更不要と見込むが、実装セッションで再確認する）。
